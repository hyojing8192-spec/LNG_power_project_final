# LNG발전 최적 가이던스 대시보드 — 화면 명세서

> **파일**: `scripts/LNG_project_final.py`  
> **프레임워크**: Streamlit + Plotly  
> **자동 새로고침**: 1분 간격 (`streamlit_autorefresh`)

---

## 1. 전체 구조 개요

```
[페이지 헤더]
  ⚡ LNG발전 최적 가이던스 제공 프로그램
  (스케줄러 상태 배지 — 우측 상단 고정, 깜빡임 애니메이션)

[사이드바]
  └─ 설정 패널

[메인 영역]
  ├─ 종합화면 (항상 표시)
  │   ├─ 제목 카드
  │   ├─ 핵심 지표 4개 (KPI 카드)
  │   ├─ 종합 시계열 차트 (SMP / BEP / LNG가격)
  │   └─ 날짜쌍 루프: 야간 테이블 + 주간 테이블
  │
  ├─ [Expander] 📈 경제성 분석 (상세)
  ├─ [Expander] 📋 가동 가이던스 (상세)
  ├─ [Expander] 🔍 이상구간 탐지
  ├─ [Expander] 🤖 ML 모델 성능
  └─ [Expander] 📁 원시 데이터

[푸터]
```

## 2. 사이드바

| 컴포넌트 | 설명 |
|---|---|
| `date_input` 분석 기준일 | 달력 날짜 선택. 주말/공휴일 선택 시 직전 영업일로 자동 보정 |
| 대상 날짜 캡션 | 기준일(D) + D+1, 연속 휴일이면 다음 영업일까지 표시 |
| `number_input` LNG 가격 | $/MMBtu, 기본값 `DEFAULT_LNG_PRICE`, step 0.5 |
| `checkbox` Spot LNG | 체크 시 제세금 0.8$/MMBtu 추가 적용 |
| LNG 열량 캡션 | 학습데이터 평균값 자동 산출 (Mcal/Nm³) |
| 환율 캡션 | 전일 평균 자동 산출 (원/$) |
| SMP 수집 상태 캡션 | 수집 완료 날짜 범위 + 날짜별 🟢/🔴 상태 |

**날짜 보정 로직:**
- 주말 또는 법정공휴일(`LEGAL_HOLIDAYS`) 선택 시 경고 메시지 + 직전 영업일로 보정
- 최신 경제성분석 CSV 날짜를 자동 감지해 기본값으로 설정 (미래 날짜 CSV는 무시)

## 3. 스케줄러 상태 배지

우측 상단에 고정(`position: fixed`) 표시. 1.4초 주기로 깜빡임.

| 상태 | 색상 | 텍스트 |
|---|---|---|
| `running` | 초록 `#28a745` | 스케줄러 정상 가동중 |
| `fetching` | 주황 `#FF8C00` | SMP 데이터 수집중 |
| `stopped` | 빨강 `#dc3545` | 스케줄러 미실행 |

- Windows 전용 (Linux/Streamlit Cloud에서는 숨김)
- `wmic`으로 `run_scheduler.py` 프로세스 확인, `XPlatform.exe` 감지 시 `fetching`

## 4. 종합화면 (메인, 항상 표시)

### 4-1. 제목 카드
- 파란 그라데이션 배경 카드 (`#2F5597` → `#4472C4`)
- 텍스트: `LNG발전 가동 경제성 판단 결과` + `M월 D일(요일) 기준`

### 4-2. 핵심 지표 KPI (4열)
| 열 | 지표 | 단위 |
|---|---|---|
| 1 | LNG 가격 (Spot/사용단가) | $/MMBtu |
| 2 | 환율 | 원/$ |
| 3 | 평균 SMP | 원/kWh |
| 4 | 최적 운전모드 (최빈값) | — |

### 4-3. 종합 시계열 차트

**차트 구성**: Plotly 이중 Y축 (Subplot secondary_y)

| 트레이스 | 타입 | Y축 | 색상 |
|---|---|---|---|
| SMP (원/kWh) | Scatter 꺾은선 | 주축 (좌) | `#2F5597` |
| LNG발전 BEP ($/MMBtu) | Bar 막대 | 보조축 (우) | `#B4C7E7` |
| LNG가격 ($/MMBtu) | Scatter 점선 | 보조축 (우) | `#ED7D31` |

**X축 범위**: D일 22시 ~ 마지막 대상 날짜 21시 (날짜 경계 수직선 표시)  
**높이**: 450~600px (데이터 포인트 수에 따라 동적)

### 4-4. 가동계획 테이블 (날짜쌍 루프)

대상 날짜 쌍(D일, D+1일, ...)마다 야간/주간 섹션이 반복 생성된다.

#### 야간 테이블 (D일 22시 ~ D+1일 08시)
- 헤더: `야간 M월D일(요일) 22시 ~ M월D일(요일) 08시`
- 컬럼 (열=시간, 행=항목):

| 행 항목 | 내용 |
|---|---|
| 최적운전모드 | 2기 full / 2기 저부하 / 1기 full / 정지 |
| SMP(원/kWh) | 시간별 실측값 |
| 수전단가(원/kWh) | 한전 수전단가 |
| 대체단가(원/kWh) | 모드별 혼합 단가 |
| LNG발전 BEP($/MMBtu) | 손익분기점 |
| 경제성(억원) | 시간별 경제성 |

**행 색상:**
- 2기 full → 파란 배경 `#D6E4F0`
- 2기 저부하 → 초록 배경 `#E2EFDA`
- 1기 full → 노란 배경 `#FFF2CC`
- 정지 → 빨간 배경 `#F8D7DA`

#### 주간 테이블 (D+1일 08시 ~ 22시)
- 구성 동일, D+1일 SMP 기반으로 별도 예측/가이던스 생성

## 5. Expander: 📈 경제성 분석 (상세)

**기본 상태**: 접힘

### 5-1. KPI 지표 (4열)
- 평균 SMP / LNG발전 BEP 임계 SMP / 기력발전 BEP 임계 SMP / 최다 최적모드

### 5-2. SMP 시계열 차트
- 24시간 SMP 꺾은선

### 5-3. 상세 테이블
- `hourly_df` 전체 (24행 × 전체 컬럼), 소수점 2자리
- CSV 다운로드 버튼 (`경제성분석_YYYY-MM-DD.csv`)

## 6. Expander: 📋 가동 가이던스 (상세)

**기본 상태**: 접힘

### 6-1. 상단 요약 바
- 날짜(요일) | LNG가격 | 환율 | 열량 한 줄 텍스트

### 6-2. KPI 지표 (4열)
- 평균 SMP / 최다 최적모드 / 일일 경제성 / 이상구간 시간수

### 6-3. 종합 운전 권고
- `[긴급]` → `st.error` (빨간)
- `[주의]` → `st.warning` (노란)
- `[참고]` → `st.info` (파란)
- 기타 → `st.success` (초록)

### 6-4. 주간 가이던스 (08~22시)
- 구간 요약 텍스트 (운전모드 분포 / 평균·최대·최소 SMP / 정지·감발·기력점화 경고)
- 가동계획표 (시간별 행 색상: 정지=빨강, 감발=노랑, 기력점화=파랑)
- 차트: SMP 꺾은선 + 최적모드 BEP 막대 + LNG가격 점선 (이중 Y축)

### 6-5. 야간 가이던스 (22시 ~ 익일 08시)
- 주간과 동일 구성 (22, 23, 00~07시 순)

### 6-6. 일일 경제성 요약
- 모드별 분포 텍스트 + 이상구간 목록 (좌)
- 모드별 경제성 막대그래프 (우)

### 6-7. 다운로드
- 가이던스 텍스트 리포트 (`.txt`)
- 가동계획표 CSV (`.csv`)

### 6-8. 카카오톡 전파 메시지
- `st.code` 블록으로 복사 가능한 형태로 표시

## 7. Expander: 🔍 이상구간 탐지

**기본 상태**: 접힘

### 7-1. KPI (3열)
- SMP 제로 건수 / SMP 경제성 한계 건수 / SMP 과대 건수

### 7-2. SMP 이상구간 차트
- `build_smp_chart()` — 전체 기간 SMP + 이상구간 하이라이트
- 임계선 2개 (LNG발전 BEP, 기력발전 BEP)

### 7-3. 이상구간 상세 테이블
- `detect_smp_anomalies()` 결과 DataFrame
- 감지 없으면 `st.success` 표시

### 7-4. 경제성 급변 탐지
- `detect_econ_change()` + `build_econ_change_chart()` 사용
- `경제성차이_2기` 컬럼 기반 급변 구간 탐지 및 시각화

## 8. Expander: 🤖 ML 모델 성능

**기본 상태**: 접힘

### 8-1. 데이터 분할 정보 (3열)
- 전체 / 학습 / 테스트 데이터 행수

### 8-2. 모드별 성능 테이블
- 운전모드 × 타깃(export/import/efficiency)별 MAE, R², CV R², Test MAE, Test R²
- R² 컬럼에 Red-Yellow-Green 컬러맵 배경

### 8-3. CV R² 막대그래프
- 운전모드별 grouped bar (export / import / efficiency 3색)

### 8-4. 모델 재학습 버튼
- 클릭 시 `retrain()` 실행 → `st.cache_resource.clear()` → `st.rerun()`

## 13. 조건부 화면 분기

| 조건 | 표시 |
|---|---|
| `data_loaded=True` + SMP 있음 | 종합화면 전체 + 모든 Expander |
| `data_loaded=True` + SMP 없음 | 빨간 "산출불가" 카드 |
| `data_loaded=False` | `st.warning` 경고 메시지 |
