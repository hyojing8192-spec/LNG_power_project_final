## 문제 정의

---

# 현황: LNG 발전소의 의사결정 구조

LNG 복합화력 발전소는 매 시간마다 **가동/미가동** 결정이 필요합니다.

- 발전소 전체 전력의 **약 90%는 자가소비**, 나머지 10%만 잉여 판매
- 전력 시장가격(SMP)이 **LNG 발전 원가(BEP)보다 높으면** → 가동 수익
- SMP가 BEP보다 낮으면 → **기력 발전으로 전환하거나 정지**

> **핵심 질문**: 오늘 각 시간대에 LNG로 가동할까? 기력으로 전환할까?

---

# 문제: 수동 분석의 한계

**기존 프로세스** (매일 반복)

```
SMP 확인 → 단가 확인 → BEP 계산 → 운전 계획 수립 → 가이던스 작성
```

총 **6~7단계** 수작업, 30~40분 소요

---

### 특히 어려운 상황들

| 상황 | 문제 |
|------|------|
| SMP 급등 (170원↑) | 기력 발전 전환 타이밍 판단 어려움 |
| SMP 급락 (0원 근처) | 손실 구간 조기 식별 불가 |
| 금요일·연휴 전날 | 다음 주 전체 계획을 한번에 작성해야 함 |
| LNG 가격 변동 | 임계값이 매일 달라져 단순 비교 불가 |

---

# 우리가 원하는 것

**세 가지 자동화 목표**

1. **SMP 공시 후 자동 분석**  
   매일 17:30 이후 SMP 확정 → 즉시 BEP 계산 → 메일·카톡 발송

2. **LNG 가격 반영 동적 임계값**  
   고정 100원/170원이 아닌, 실시간 LNG 가격·환율 기반 BEP 자동 산출

3. **ML 예측으로 운전 파라미터 자동화**  
   역송량·수전량·효율을 XGBoost로 예측 → 운전 모드별 정밀 BEP 계산

# F1. SMP 수집 — 3단계 폴백 구조

KPX는 매일 **17:30 전후** SMP를 공시합니다.

| 우선순위 | 수집 방법 | 특징 |
|----------|-----------|------|
| **1순위** | ePower KMOS GUI 자동화 (PyAutoGUI RPA) | 가장 빠름, 공식 엑셀 |
| **2순위** | KPX 웹 크롤링 (BeautifulSoup4) | 서버 환경에서도 동작 |
| **3순위** | 공공데이터포털 API | 최종 폴백 |

- 수집 실패 시 **10분 간격 자동 재시도** (최대 19:00까지)
- 결과를 `smp_YYYY-MM-DD.json`으로 캐싱

> ePower KMOS는 공개 API 없음 → PyAutoGUI로 GUI 직접 조작

---

# F2. ML 예측 — XGBoost 회귀

운전 모드(1기 / 2기 / 저부하)별로 **역송량, 수전량, 효율**을 예측

### 주요 피처

| 그룹 | 피처 |
|------|------|
| 시장 | SMP, LNG가격, 환율, 전기요금 |
| 운전 | 역송량, 수전량, LNG 열량 |
| 시계열 | 시간 sin/cos, 월 sin/cos, 요일, 월 |
| 구분 | 저부하 여부, 고효율 계절 여부 |

- **Stratified K-Fold** 교차검증으로 시계열 특성 반영
- 소수 운전모드 → **SMOTE** 오버샘플링으로 불균형 처리

---

# F3. 경제성 분석 — BEP 역산

**손익분기 SMP(BEP) = LNG 연료비 ÷ 열효율**

| 발전 방식 | 열효율 | BEP 계산 |
|-----------|--------|----------|
| LNG 1기 | 1.592 Mcal/kWh | LNG가격 × 환율 × 열효율 ÷ 860 |
| LNG 2기저부하 | 1.679 Mcal/kWh | ↑ 동일 방식 |
| LNG 2기 | 1.574 Mcal/kWh | ↑ 동일 방식 |
| 기력 발전 | 2.3 Mcal/kWh | SMP가 높을 때만 유리 |

- **SMP ≥ BEP** → LNG 가동 권장
- **SMP ≥ 기력 BEP** → 기력 전환 검토
- Spot 계약 시 +0.8$/MMBtu 제세금 반영

---

# F4. 이상치 탐지 — 동적 임계값

고정 임계값(100원, 170원)이 아닌 **당일 LNG가격·환율 기반 자동 산출**

```
smp_low  = LNG 발전 BEP  (이 이하 → 가동 손실)
smp_high = 기력 발전 BEP (이 이상 → 기력 전환 필요)
```

### 3단계 분류

| 구분 | 조건 | 조치 |
|------|------|------|
| 🟢 정상 | smp_low ≤ SMP ≤ smp_high | 유지 |
| 🟡 주의 | SMP < smp_low | 감발 또는 정지 검토 |
| 🔴 위험 | SMP > smp_high | 기력 전환 즉시 검토 |

- 이상구간 발생 시 **긴급 알림 메일** 자동 발송

# 개발 과정의 주요 도전

### 1. SMP 수집 안정성
- KPX 크롤링만으로는 주말·공휴일 미공시 대응 불가  
  → **ePower GUI 자동화(RPA) + 3단계 폴백 구조** 도입

### 2. 춘추계절 전기요금 특례 처리
- 11~13시 50% 할인 구간 → 단순 '경부하' 표시로는 BEP 오계산  
  → **시간대별 실제 요금을 BEP와 함께 표시**, 담당자가 직접 확인 가능하도록

### 3. 다음날 SMP 날짜 경계 처리
- 발송 시점(D일)과 표시 날짜(D+1)가 혼재 → 화면에 익일 SMP가 잘못 표시  
  → **D일/D+1 명확히 구분**, 공시 여부 플래그로 조건부 표시

# 머신러닝 관련 인사이트

**처음 예상과 달랐던 점**

- ML 학습 초기: R² 지표가 낮아 모델이 안 되는 것 같았음  
  → 실제 원인은 **SMOTE 처리 후 불균형 구간에서 과적합** → K-Fold 재설계로 개선

- 5번 학습 후 특정 날짜에서 예측 오차가 컸는데  
  → 테스트 셋 설계에서 **해당 날짜가 고부하 특수일임을 사전 확인할 수 있었을 것**

**XGBoost를 선택한 이유**
- 시계열 특성을 피처로 인코딩하면 테이블형 모델로도 충분
- 학습 속도 빠름, 피처 중요도 해석 가능
- LSTM 등도 검토했으나 데이터 양 대비 과사양 판단

# 예상 Q&A

**Q. XGBoost를 선택한 이유는?**  
시계열 피처를 sin/cos로 인코딩하면 테이블형 모델로도 충분하고, 학습 속도·해석 가능성에서 유리합니다. LSTM도 검토했으나 데이터 규모 대비 과사양으로 판단했습니다.

**Q. LNG 가격 변동 시 임계값이 얼마나 달라지나요?**  
LNG 8$/MMBtu → 약 100원, 14$/MMBtu → 약 175원 수준으로 임계값이 달라집니다. 고정 임계값으로는 이를 반영할 수 없어 동적 계산이 필수입니다.

**Q. SMP 데이터를 못 받으면 어떻게 되나요?**  
3단계 폴백 + 10분 재시도(19:00까지)로 대응하며, 모두 실패 시 전날 데이터 폴백 후 로그에 기록합니다. 잘못된 데이터로 잘못된 가이던스를 내는 것을 방지합니다.

# 예상 Q&A (2)

**Q. SMOTE는 어떻게 적용했나요?**  
SMOTE는 학습 데이터에만 적용하고 테스트 데이터에는 적용하지 않습니다. Stratified K-Fold로 교차검증하여 과적합을 모니터링하고, MIN_R² 기준 미달 시 재학습하는 안전장치를 두었습니다.

**Q. 기력 발전 BEP는 왜 별도로 계산하나요?**  
기력발전은 LNG와 연료가 달라 열효율이 다릅니다(2.3 Mcal/kWh). SMP가 매우 높을 때 기력 발전으로 수익 창출 기회가 생기므로, 두 BEP를 모두 표시해야 정확한 운전 전략을 세울 수 있습니다.
